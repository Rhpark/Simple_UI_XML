# Step 1. 기능 검증

요청된 기능에 맞게 코드가 작성되었는가?

## 체크리스트

- [ ] 요청된 기능이 실제로 구현되었는가?
- [ ] 기능 기준 문서(PRD/SPEC/IMPLEMENTATION_PLAN)를 확인했는가?
- [ ] 엣지 케이스가 처리되었는가?
- [ ] 기능 누락이 없는가?
- [ ] 과잉 구현(요청하지 않은 기능 추가)이 없는가?
- [ ] 입력값 유효성 검사가 필요한 경우 처리되었는가?

## 심각도 기준

- CRITICAL: 핵심 기능이 구현되지 않음
- HIGH: 엣지 케이스 미처리로 런타임 오류 가능성
- MEDIUM: 과잉 구현 또는 불필요한 기능 포함
