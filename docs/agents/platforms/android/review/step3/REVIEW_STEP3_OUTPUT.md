## 산출물

STEP3 본 산출물 `[아키텍처]` 블록 바로 아래에 `[Android 아키텍처]` 블록을 평행 블록으로 추가한다.
본 블록은 STEP4로 그대로 전달된다.

STEP3 본 산출물의 다음 항목에 Android 결과를 1줄씩 반영한다:
- `레이어 확인`의 `DAO 직접 호출` 항목
- `경계 확인`의 `모듈 경계` 항목
- `설계 원칙 확인`의 `SRP` / `DIP` 항목

`[Android 아키텍처]` 블록 형식:

```text
[Android 아키텍처]
레이어 건너뜀  : [없음 (확인 완료) 또는 위반 위치]
DAO 직접 호출  : [없음 (확인 완료) 또는 위반 위치]
UI 직접 변경   : [없음 (확인 완료) 또는 위반 위치]
DI 패턴        : [Hilt / Koin / 수동 / 없음]
DIP            : [DI 컨테이너 주입 확인 — 위반 없음 또는 위반 위치]
Context 주입   : [적합 또는 위반 위치 또는 "해당 없음"]
SRP            : [적합 또는 위반 위치 또는 "해당 없음"]
가변 상태 노출 : [없음 (확인 완료) 또는 위반 위치]
ViewModel 변경 : [없음 (확인 완료) 또는 위반 위치 또는 "해당 없음"]
멀티 모듈      : [단일 모듈 — 해당 없음 또는 위반 없음 또는 위반 위치]
Manifest 매칭  : [일치 (확인 완료) 또는 불일치 위치]
위반 요약      : [위반 건수] 건 (없으면 "없음")
```
