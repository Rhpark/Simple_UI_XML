## 산출물

STEP1 본 산출물 `호환성` 줄 바로 아래에 `[Android 호환성]` 블록을 평행 블록으로 추가한다.
본 블록은 STEP2 → STEP3 → STEP4로 그대로 전달된다.

`호환성` 줄 1줄 요약 형식:
- API 사용 있음: `호환성 : Android — 감지: [카테고리 요약] / 위반: [건수 또는 "없음"] (상세: [Android 호환성] 블록 참조)`
- API 사용 없음: `호환성 : Android — 해당 없음 (Android 프레임워크 API 미사용)`

`[Android 호환성]` 블록 형식:

```text
[Android 호환성]
감지 카테고리 : [카테고리명 목록 또는 "해당 없음 — Android 프레임워크 API 미사용"]
minSdk        : [프로젝트 minSdk 값]
SDK 대조      : [API명 — 최소 요건 — 충족 여부] (줄 단위 나열, 없으면 "해당 없음")
권한 확인     : [권한명 — 매니페스트 OK/누락 — 런타임 체크 OK/누락] (없으면 "해당 없음")
Foreground Service : [감지 여부 / 타입 / 매니페스트 정합성 또는 "해당 없음"]
저장소        : [Scoped Storage 정합성 또는 "해당 없음"]
PendingIntent : [FLAG 명시 여부 또는 "해당 없음"]
EdgeToEdge    : [적용 여부 또는 "해당 없음"]
위반 요약     : [위반 건수] 건 (없으면 "없음")
```

사용자 검증 포인트:
- `감지 카테고리`가 "해당 없음"이면 실제로 Android 프레임워크 API가 없는지 import 목록으로 직접 확인했는가.
- `SDK 대조`에서 충족 여부가 "미충족"인 항목의 라인번호가 실제 코드에 존재하는가.
- `권한 확인`에서 매니페스트 누락 항목이 실제로 `AndroidManifest.xml`에 없는가.
