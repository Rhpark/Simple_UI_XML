# STEP_ROLLBACK.md — 루프 판정 및 복귀 규칙

이 파일은 analysis·review 두 워크플로우가 공유하는 루프 판정 및 복귀 규칙이다.

---

## 루프 판정 기준

아래 두 조건을 모두 충족하면 루프로 판정한다.

1. 동일한 STEP에 복귀한 횟수가 **2회 이상**이다.
2. 복귀 원인이 직전 복귀와 **동일**하다 — 같은 하네스 항목 미통과 또는 같은 강제 중단 조건.

### 루프 판정 시 행동

즉시 작업을 멈추고 사용자에게 아래 내용을 보고한다.

```text
루프 판정 보고
발생 STEP    : STEP N
반복 원인    : 미통과 항목명 또는 강제 중단 조건명
복귀 횟수    : N회
현재 산출물  : 완성된 항목 목록 (미완성 항목은 "미완성")
사용자 선택지:
  1. 분석·리뷰 범위를 재정의하고 STEP1부터 재시작
  2. 해당 항목을 수동으로 입력해 진행
  3. 작업 취소
```

---

## 복귀 조건 및 대상 STEP

복귀는 원인이 명시된 STEP으로만 한다. 임의로 더 앞 STEP으로 건너뛰지 않는다.

| 복귀 원인 | 복귀 대상 |
|---|---|
| 진입점 정보 재확인, 핵심 파일 재확정, 분석 타입 재분류 | STEP1 |
| 근거 파일·호출 흐름·문서 주장 재추적 | STEP2 |
| 영향 계층·직접 파일·간접 파일 재확인 | STEP3 |
| 사실/추정 분리 불가, 근거 부족 | STEP2 또는 STEP3 |
| 최종 보고 단계에서 핵심 파일 누락 확인 | STEP1 |

---

## 부분 복귀 원칙

- 복귀 시 해당 STEP의 산출물만 재작성한다.
- 그 이전 STEP의 산출물은 변경하지 않는다.
- 단, 복귀 원인이 이전 STEP의 오류임이 명확히 확인된 경우에는 해당 STEP까지 역으로 복귀한다.
