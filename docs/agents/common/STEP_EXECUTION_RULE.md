# Step Execution Rule

이 파일은 analysis·review 두 워크플로우의 모든 STEP에 공통으로 적용되는 실행 규칙이다.
각 워크플로우의 에이전트는 **세션 시작 시 이 파일을 먼저 읽는다.**

---

## 기본 원칙

| # | 원칙 | 설명 |
|---|---|---|
| 1 | **STEP 파일 선행 읽기** | 각 STEP 시작 전 해당 STEP 파일을 연다. 기억에 의존하지 않는다. |
| 2 | **체크리스트 = 작업 큐** | 실행 체크리스트를 순서대로 완료한다. 항목을 건너뛰지 않는다. |
| 3 | **하네스 통과 전 이동 금지** | 현재 STEP 하네스를 통과하지 못하면 다음 STEP으로 이동하지 않는다. |
| 4 | **사실 기준** | 직접 열어 확인한 코드·문서·실행 결과만 사실로 기록한다. |
| 5 | **추정 분리** | 근거 없이 추론한 내용은 반드시 추정으로 분류하고 사실과 섞지 않는다. |

---

## 사실 기록 기준

- **사실**: 파일명:라인번호 또는 공식 문서 URL 근거가 있는 항목
- **추정**: 근거 없이 추론했거나, 열지 않은 파일에 대한 판단
- 작성 후 사실 항목 각각에 파일명:라인번호가 있는지 재확인한다.
- 근거가 없는 항목은 추정으로 이동한다.

---

## 미확인·부재 항목 표기 규칙

| 상황 | 표기 |
|---|---|
| 해당 항목이 존재하지 않음을 직접 확인 | `없음 (확인 완료)` |
| 확인 필요하나 아직 못 한 경우 | `미확인 — 이유` |
| 해당 입력 타입 자체가 존재하지 않거나 도달 불가 경로 | `해당 없음 — 이유` |

모호하거나 불확실한 항목은 추측으로 채우지 않고 사용자에게 질문한다.

---

## STEP 간 데이터 전달 규칙

- 이전 STEP 산출물 전체를 다음 STEP의 입력으로 전달한다.
- 산출물 텍스트를 그대로 포함한다. 요약·생략하지 않는다.
- 진입 조건 항목 중 하나라도 비어 있으면 이전 STEP으로 복귀해 누락 항목을 채운다.
- 이전 STEP 산출물은 수정하지 않는다. 현재 STEP 산출물만 새로 작성한다.

---

## 하네스 실패 시 행동 흐름

```
하네스 항목 미통과
  │
  ├─ CRITICAL → 즉시 중단 또는 해당 STEP 복귀
  ├─ HIGH     → 해당 항목 재수행 후 진행
  ├─ MEDIUM   → 리스크 기록 후 조건부 진행
  └─ LOW      → 기록 후 계속 진행
```

심각도 정의 → [SEVERITY_RULE.md](SEVERITY_RULE.md) 참조
복귀·루프 판정 → [STEP_ROLLBACK.md](STEP_ROLLBACK.md) 참조 / 강제 중단 → [STEP_FORCE_STOP.md](STEP_FORCE_STOP.md) 참조 / 하네스 실패 → [STEP_HARNESS.md](STEP_HARNESS.md) 참조

## 우선순위 규칙

체크리스트는 최소 보장선이지 상한선이 아니다.
결론을 바꿀 수 있는 사실을 발견하면 체크리스트 외 항목이라도 이슈·리스크에 기록한다.
