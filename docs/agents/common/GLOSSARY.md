# Glossary

이 파일은 analysis·review·feature 세 워크플로우가 공유하는 공용 용어 정의다.
워크플로우 전용 용어는 각 워크플로우의 `common/GLOSSARY.md`를 참조한다.

---

## 파일 분류 용어

| 용어 | 정의 |
|---|---|
| **핵심 파일** | STEP1에서 분석 시작점으로 선정한 파일. 결론에 직접 영향을 주는 파일만 포함한다. 로직 없는 래퍼·단순 데이터 홀더·자동 생성 파일은 제외한다. |
| **근거 파일** | STEP2 이후 실제로 열어 확인한 파일 또는 공식 문서 URL. 반드시 파일명:라인번호로 기록한다. |
| **직접 파일** | 분석 결론 또는 후속 조치의 직접 대상. 이번 결론을 바꾸거나 후속 조치에서 직접 다룰 파일. |
| **간접 파일** | 직접 파일 때문에 함께 영향을 받는 대상. 호출 대상, 테스트, 의존 문서를 포함한다. |
| **공통 파일** | 여러 기능이나 계층이 공유하는 기반 파일. util, base, 공유 유틸리티, DI 모듈 등. |

> 같은 파일이 직접·간접 두 목록에 해당하면 직접으로만 기록한다.

---

## 범위 용어

| 용어 | 정의 |
|---|---|
| **영향 범위** | 분석 결론 또는 후속 조치로 동작·문서·검증 범위가 달라질 수 있는 전체 범위. |
| **읽은 범위** | STEP2에서 실제로 열어 확인한 파일·문서의 집합. |
| **제외 범위** | STEP1에서 명시적으로 추적 경계로 설정한 파일·모듈·시나리오. |
| **미확인 범위** | 분석에 필요하나 아직 확인하지 못한 파일·조건·환경. |

---

## source of truth (SSOT)

단일 정보 출처. 같은 기준이 여러 문서에 중복 정의되면 충돌이 발생한다.
각 기준은 하나의 파일에만 정의하고, 다른 파일은 링크로만 참조한다.

---

## 산출물 상태 표현

| 표현 | 의미 |
|---|---|
| `없음 (확인 완료)` | 해당 항목이 존재하지 않음을 직접 확인했다. |
| `미확인 — 이유` | 확인이 필요하나 해당 이유로 아직 확인하지 못했다. |
| `해당 없음 — 이유` | 이 항목의 입력 타입 자체가 존재하지 않거나 도달 불가능한 경로다. |
| `미완성` | 진행 중이거나 하네스 미통과로 완료되지 않은 항목. |
