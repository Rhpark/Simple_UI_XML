두 워크플로우 공용 용어는 [../../common/GLOSSARY.md](../../common/GLOSSARY.md)를 참조한다.

## STEP 간 전달 필드

### [분석 입력] — analysis STEP4 carry (모든 STEP에 전달)

| 필드 | 정의 |
|---|---|
| `직접 파일` | analysis STEP4 `[분석 요약] 직접 파일` 그대로. 직접 관련 파일 목록. 파일명:라인번호 형식. |
| `영향 계층` | analysis STEP4 `[분석 요약] 영향 계층` 그대로. 계층 정보. |
| `기준 문서` | analysis STEP4 `[분석 요약] 기준 문서` 그대로. 파일명:라인번호 또는 "없음/미확인". |

### [기능 검증] — STEP1 산출물

| 필드 | 정의 |
|---|---|
| `검증 대상` | 검증한 기능명 또는 시나리오명. analysis `[분석 요약] 핵심 파일`의 진입점 기준. |
| `검증 맥락` | 분석 요약(analysis `한 줄 결론`), 기준 문서([분석 입력] `기준 문서`), 확인 범위(analysis `범위`)를 포함한 검증 맥락. |
| `엣지 케이스` | 엣지 케이스 5종 확인 결과. null/empty, 0개 데이터, 단일 항목, 최대 개수, 실패 응답. |
| `발견 사항` | STEP1에서 발견한 기능 관련 finding 목록. 없으면 "없음". |
| `이관 항목` | STEP1에서 발견했지만 STEP2·STEP3·STEP4 소관으로 넘긴 항목 목록. |

### [로직 & 안정성] — STEP2 산출물

| 필드 | 정의 |
|---|---|
| `로직 확인` | 로직 오류 4종 확인 결과. 조건 분기, 무한 루프, null 처리, 경계값. |
| `런타임 확인` | 런타임 안정성 4종 확인 결과. 메모리 누수, 비동기 누수, 생명주기 위반, 예외 처리. |
| `발견 사항` | STEP2에서 발견한 로직·안정성 finding 목록. 없으면 "없음". |
| `확인 범위` | STEP2 확인 범위 / 제외 범위 / 미확인 런타임 범위. |
| `이관 항목` | STEP3·STEP4 소관으로 넘긴 항목 목록. |

### [아키텍처] — STEP3 산출물

| 필드 | 정의 |
|---|---|
| `기준 문서` | 아키텍처 기준 문서 또는 공식 기준. 없으면 "없음", 미확인이면 "미확인 — 이유". |
| `레이어 확인` | 레이어 책임 분리 3종 확인 결과. 레이어 건너뜀, DAO 직접 호출, UI 직접 변경. |
| `경계 확인` | 모듈 경계·가변 상태 노출 확인 결과. |
| `설계 원칙 확인` | SRP·DIP 확인 결과. |
| `발견 사항` | STEP3에서 발견한 아키텍처 finding 목록. 없으면 "없음". |
| `확인 범위` | STEP3 확인 범위 / 제외 범위 / 미확인 아키텍처 범위. |
| `이관 항목` | STEP4 소관으로 넘긴 항목 목록. |

### [코드 품질] — STEP4 산출물

| 필드 | 정의 |
|---|---|
| `도구 실행` | 실행한 도구와 결과 또는 미실행 이유. |
| `품질 확인` | 코드 품질 5종 확인 결과. 미사용 코드, 중복 로직, deprecated API, hallucination, 주석 코드. |
| `컨벤션 확인` | 컨벤션 3종 확인 결과. 네이밍, 로깅 패턴, 버전 분기 패턴. |
| `컨벤션 기준` | 컨벤션 기준 문서. 없으면 "컨벤션 문서 없음 — 확인 불가". |
| `발견 사항` | STEP4에서 발견한 품질 finding 목록. 없으면 "없음". |
| `확인 범위` | STEP4 확인 범위 / 제외 범위 / 미확인 품질 범위. |
| `상위 이관` | STEP1/STEP2 대표로 이관한 항목. 없으면 "없음". |

### [리뷰 요약] / [리뷰 보고] — STEP5 산출물

| 필드 | 정의 |
|---|---|
| `[리뷰 요약] 한 줄 결론` | 리뷰 전체 한 줄 결론. |
| `[리뷰 요약] 점수` | Agent 수행성 / 사용자 이해성 점수. 각 100점 기준. |
| `[리뷰 요약] 감점 이유` | 각 점수의 감점 이유. findings·확인 범위·도구 실행 결과와 연결해 기록. |
| `[리뷰 요약] 확인 범위` | STEP1~4 확인 범위 병합 결과. 확인 범위 / 제외 범위 / 미확인 범위. |
| `[리뷰 요약] 우선 행동` | 병합된 findings 중 우선 처리 조치 최대 3개. finding 없으면 "없음". |
| `[리뷰 요약] 단계별 발견 수` | STEP1~4 각 단계별 finding 건수 요약. |
| `[리뷰 보고] 발견 사항` | STEP1~4 모든 finding을 중복 제거 후 심각도 순 정렬한 목록. |
| `[리뷰 보고] 중복 제거` | 중복 제거된 항목 — 원본 STEP / 대표 STEP / 제거 이유. 없으면 "없음". |
| `[리뷰 보고] 도구 실행` | 도구 실행 결과 또는 미실행 이유 요약. |
| `[리뷰 보고] 다음 단계` | 다음 권장 워크플로우. feature / refactor / 리뷰 완료 중 하나. |
