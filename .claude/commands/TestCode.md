docs/rules/TEST_RULE.md 의 테스트 규칙을 읽고,
아래 우선순위에 따라 대상 코드를 결정해서 테스트 코드를 생성해줘.

## 대상 코드 결정 우선순위
1. $ARGUMENTS 가 있으면 → 해당 파일 경로, 패키지명, 클래스명을 대상으로 한다.
2. $ARGUMENTS 가 없으면 → 현재 IDE에서 선택된 코드(ide_selection)를 대상으로 한다.

## 작업 순서
1. 대상 코드를 분석해서 Android 프레임워크 의존 여부를 판단한다.
    - 의존 있음 → Robolectric 테스트
    - 의존 없음 → Unit 테스트
2. TEST_RULE.md 규칙에 맞는 테스트 파일 경로와 파일명을 결정한다.
3. 각 함수/클래스에 대한 테스트 코드를 작성한다.
4. docs/rules/TEST_RULE.md 에 맞게 작성되었는지 검증한다.
5. 검증을 통과하면 테스트 파일을 새로 생성한다.
6. 생성된 파일 경로와 성공/실패 여부를 요약해서 알려준다.
