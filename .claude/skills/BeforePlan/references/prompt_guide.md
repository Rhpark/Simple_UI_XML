# PROMPT 작성 가이드

## 역할

PROMPT 문서는 **사용자와 주고받은 대화의 기록(로그)** 이다. BeforePlan 진행 중 실시간으로 갱신한다.
PRD/SPEC의 "근거 원본"이자 감사 추적용이며, /plan에는 전달하지 않는다.

## 핵심 규칙: append-only

- **덧붙이기만** 한다. 이미 기록한 내용을 요약·재해석·삭제·재배치하지 않는다.
- "내용 추가"는 허용된다(대화가 늘어나므로 당연). 금지되는 것은 **기존 기록의 변형**이다.
- 시간순으로 누적한다. 나중에 답이 바뀌면 이전 기록을 고치지 말고, 새 항목으로 추가한다.
  (예: "Q3 답변 정정: ..." 을 새로 덧붙임)

## 기록 대상

- 사용자 원문 요청 (최초 프롬프트 그대로)
- 명확화를 위해 던진 질문과 사용자의 답변
- 사용자가 내린 결정·선택 (선택지가 있었다면 무엇을 골랐는지)
- SPEC 작성 중 발생한 질문과 그 답변도 여기에 누적

## 형식 (예시)

```md
# PROMPT

## 최초 요청 (원문)
> {사용자가 처음 말한 내용 그대로}

## 대화 로그 (시간순 append-only)

### Q1. {질문}
- 답변: {사용자 답변}

### Q2. {질문}
- 답변: {사용자 답변}

### [SPEC 탐색 중] Q3. {질문}
- 답변: {사용자 답변}
- 영향: PRD 수정 / SPEC 수정 (어느 쪽인지 표기)
```

## 하지 말 것

- 사용자가 하지 않은 말을 기록하지 않는다.
- 추정·해석을 사용자 발언처럼 적지 않는다 (해석은 PRD에서 한다).
- 기존 항목을 덮어쓰지 않는다 (정정도 새 항목으로).
