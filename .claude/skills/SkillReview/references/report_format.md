# SkillReview 보고 형식

보고는 아래 구조를 그대로 따른다. 모든 지적은 "왜 + 어떻게"를 한 묶음으로 — 단순 지적 금지.

## 1. TL;DR (한 줄)

```
[대상 N개] | [S:○ A:○ B:○ C:○ F:○] | [BLOCKER 총 ○건 — 최우선: {요약}]
```

## 2. 판정 전제 (정직 고지)

- 적용한 환경 보정 (rubric.md 0절 중 해당 항목)
- 본문 미정독 대상이 있으면 명시 ("frontmatter·구조·참조 중심, 콘텐츠 미검증")

## 3. 등급표

| 대상 | 등급 | BLOCKER | MAJOR | 핵심 근거 (파일:라인 또는 실측) |
|------|------|---------|-------|------|

## 4. 이슈 상세 (심각도 순)

각 이슈:
```
[심각도] {제목} — {대상}
위치: {파일:라인 또는 실측 명령 결과}
왜: {방치 시 무슨 일이 생기는가 — "조용히 깨지는" 경로를 구체적으로}
어떻게: {수정 방향 1~2줄}
```

## 5. 우선 조치 3분류

- **즉시 수정 (BLOCKER)**: ...
- **권장 수정 (MAJOR)**: ...
- **허용 가능 (MINOR/트레이드오프)**: 사유와 함께 — 수정하지 않는 것도 결정이다

## 6. 마무리

- 수정 진행 여부를 사용자에게 확인 (이 스킬은 읽기 전용 — 수정은 승인 후 별도 진행)
- 수정 후에는 해당 대상만 재점검(증분)을 제안
