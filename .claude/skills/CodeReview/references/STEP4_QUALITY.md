# Step 4. 코드 품질 & 성능

불필요한 코드가 없는가? 성능상 문제가 없는가?

## 체크리스트

### 품질
- [ ] 미사용 import가 없는가?
- [ ] 미사용 변수, 함수가 없는가?
- [ ] 중복 로직이 없는가?
- [ ] deprecated API를 사용하지 않는가?
- [ ] 존재하지 않는 API/클래스/메서드를 사용하지 않는가? (hallucination 방지)
- [ ] 주석 처리된 코드가 남아있지 않은가?

### 성능
- [ ] 메인 스레드에서 무거운 I/O·연산을 수행하지 않는가?
- [ ] 반복 경로에서 불필요한 객체 할당/오토박싱이 없는가?
- [ ] 반복문·컬렉션 처리에서 중복 측정·재계산이 없는가?

## 심각도 기준

- HIGH: 존재하지 않는 API 사용 (hallucination)
- HIGH: deprecated API 신규 사용
- HIGH: 메인 스레드에서 보장된 블로킹 I/O (무조건 메인 스레드에서 실행)
- MEDIUM: 호출 컨텍스트에 따라 블로킹 가능한 I/O (suspend인데 Dispatchers 미지정 등)
- MEDIUM: 미사용 코드, 중복 로직
- MEDIUM: 반복 경로의 불필요한 할당/재계산
