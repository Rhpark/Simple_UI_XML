# INTERACTION_RULE — Sub-agent 사용자 상호작용 규칙

이 파일은 analysis·review 에이전트가 sub-agent로 호출될 때 사용자 질문 처리와 재진입 동작을 정의한다.

## 실행 방식에 따른 차이

| 실행 방식 | 사용자 직접 질문 | 설명 |
|----------|--------------|------|
| Agent 도구로 호출 (nested) | 불가 | 반환값 하나만 메인 에이전트로 전달됨 |
| Claude Code 자동 선택 (최상위) | 가능 | 메인 대화 에이전트로 직접 실행됨 |

## Nested 호출 시 질문 처리 흐름

질문이 필요한 상황이 발생하면 현재 호출을 종료하고 반환값에 질문을 명시한다.
메인 에이전트가 사용자에게 전달하고, 답변을 포함해 재호출한다. 이때 새 인스턴스가 생성된다(일시중단이 아님).

```
인스턴스 A → [사용자 확인 필요] + 완료된 STEP 산출물 전부 포함 → 종료
메인 에이전트 → 사용자에게 질문 전달
사용자 답변 → 메인 에이전트 → 인스턴스 B 재호출 (이전 산출물 + 답변 포함)
인스턴스 B → STEP 식별 레이블 감지 → 중단 지점부터 재개 (분기 D)
```

## 질문 반환 형식

현재 호출 종료 시 완료된 STEP 산출물을 전부 포함하고 아래 블록을 마지막에 추가한다.

```text
[사용자 확인 필요]
확인 항목 : {질문 내용}
이유      : {이 정보가 필요한 이유}
중단 지점 : STEP{N} — {완료된 내용 요약}
```

## 재진입 동작 (분기 D)

재호출된 인스턴스는 아래 순서로 재진입 여부를 판단한다.

1. 컨텍스트에 이전 STEP 산출물 블록이 있는지 확인한다.
   - analysis: `[analysis workflow — STEP{N} 산출물]` 레이블
   - review: `[기능 검증]` / `[로직 & 안정성]` / `[아키텍처]` / `[코드 품질]` / `[리뷰 보고]` 레이블
     또는 `현재 단계: 리뷰 - STEP{N}` 헤더
2. 가장 마지막으로 완료된 STEP 블록을 확인한다.
3. 그 다음 STEP부터 재개한다. 완료된 STEP은 재수행하지 않는다.
4. review STEP5/6의 경우 `docs/agents/output/review/` 저장 파일이 있으면 파일에서 산출물을 복원한다.

## "사용자에게 질문한다" 해석 기준

- **Nested 실행**: [질문 반환 형식]에 따라 반환값에 포함하고 현재 호출을 종료한다.
  메인 에이전트가 재호출하면 새 인스턴스가 분기 D로 이어간다.
- **최상위 실행**: 직접 질문한다.

## 한계

analysis STEP1~3은 파일 저장 규칙이 없다. 재호출 시 메인 에이전트가 이전 산출물 텍스트 전체를 프롬프트에 포함해야 복원이 가능하다. 텍스트가 누락되면 해당 STEP부터 재수행한다.
