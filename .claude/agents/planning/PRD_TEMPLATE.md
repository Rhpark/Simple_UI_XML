# PRD 템플릿

## 필수 항목 구조

```md
## 문서 정보
- 문서명 / 작성일 / 대상 모듈 / 패키지 / 상태

## 배경/문제 정의
## 목표
## 비목표
## 범위
- 지원 환경 (minSdk / targetSdk)
- 기능 범위

## 설계 원칙
## API 설계 (공개 API 개요)
## SDK 분기/주의사항
## 에러 처리/로깅
## 테스트 전략
## 리스크/오픈 이슈

## 금지 패턴
- 이 기능을 수정/확장할 때 절대 하면 안 되는 것을 열거한다
- 예: "Activity/Fragment 의존 로직을 core에 두지 않는다"
- 예: "전역 싱글턴 인스턴스를 허용하지 않는다"

## 판단 기준
- A vs B 선택 상황에서 어느 것을 쓰는가를 명시한다
- 예: "Activity 없이 동작 가능한 로직 → core / Activity 필요 → xml"
- 예: "즉시 반영 → normal 패키지 / 대량 변경 → list 패키지"

## 경계 조건
- 이 기능이 책임지는 범위
- 이 기능이 책임지지 않는 범위 (비목표보다 구체적 — 코드 작성 시 판단 가능한 수준)
- 예: "권한 요청 UI(다이얼로그/토스트)는 이 기능이 제공하지 않는다"
```

## 작성 규칙

- 반드시 UTF-8 한글로 작성한다. 작성 후 한글 깨짐을 검수한다.
- 기존 문서(permissions, logcat 등) 구조를 참고해 일관성을 유지한다.
- minSdk/compileSdk/Kotlin 버전 등 환경 정보를 기준으로 기술 제약을 명시한다.
- 가정이 필요한 경우 "가정:" 으로 표시하고 근거를 함께 작성한다.
- 금지 패턴 / 판단 기준 / 경계 조건은 코드 작성 시 즉시 판단 가능한 수준으로 구체적으로 작성한다.
