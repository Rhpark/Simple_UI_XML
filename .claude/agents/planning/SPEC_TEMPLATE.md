# SPEC 템플릿

## 필수 항목 구조

```md
## 문서 정보
## 목표
## 공개 API (확정 — 함수 시그니처 포함)
## 내부 구조 (의존/비의존 분리)
## 파일/클래스 구조 (확정)
## 패키지 역할 상세
## 주요 정책 상세 (Lifecycle / 상태 보존 / 큐 등 기능별)
## 에러 처리/로깅
## 테스트 전략
## 오픈 이슈

## 판단 기준 (기능 확장 시 적용)
- 새 클래스/함수 추가 시 어느 패키지에 두는가
- core/xml 중 어디에 속하는가 판단 기준
- 기존 클래스 수정 vs 새 클래스 생성 판단 기준
```

## 작성 규칙

- 반드시 UTF-8 한글로 작성한다. 작성 후 한글 깨짐을 검수한다.
- 공개 API는 함수 시그니처(파라미터 타입, 반환 타입)까지 확정하여 작성한다.
- 내부 구조는 의존/비의존 분리 기준을 명확히 한다.
- 판단 기준은 기능 확장 시 AI가 즉시 적용할 수 있는 수준으로 구체적으로 작성한다.
